# Dracula2 Docs

This folder is the durable home for project documentation.

Use `Docs/` for:

- Codex handoffs and session summaries.
- Design plans and implementation notes.
- Workflow notes that should survive beyond one chat thread.
- Specs, pitches, and reference notes.

Use dated handoff files named like `codex-handoff-YYYY-MM-DD.md`. Keep temporary screenshots and Unity captures in `Assets/Screenshots/...`, but reference the important ones from the handoff.
